How does JavaScript make pages more interactive?

  • Access content
  • Modify content
  • React to user events
  • Recipe-like programming rules

Examples of accessing content using JS

  • Select the text inside of a <h1> element
  • Select an element that has the class of "container"
  • Find out the content of an input with the id of "email"

Examples of modifying content using JS

  • Add a paragraph after the first h1 element
  • Change the class of an element to make it display differently
  • Change the content inside of a paragraph element

We'll be using a library called jQuery to perform these actions

Examples of events we can listen to using JS

  • When a user hovers over an image
  • When a user clicks a button
  • When an amount of time has passed

Recipes are how we bring accessing, modifying, and events together.

  • When a user hovers over an image, add a paragraph elements with caption text
  • When a user clicks on a submit button, display the result
  • After every second, add 5 to the total amount and display the result

Let's see an example of listening to events and recipes

We have another word for these recipes. They're called scripts. A script is a series of instructions that a computer can follow to achieve a goal.

Unlike people, computers need very specific instructions to follow these recipes.

4 steps to making pages interactive

  • Access the content of a page
  • Modify the content of a page
  • React to events triggered by the user or page
  • Program rules or instructions to follow

Let's look at an example of all of these items coming together

Steps of our recipe

  • React: user types a letter into the input
  • Access: get the value of the input
  • Program: check if the input matches the specifications
  • Modify: if it matches, display a nice message. if it doesn't, display the error class.
